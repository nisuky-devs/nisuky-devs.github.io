Présentation des différentes balises:

Version Bêta
  • Ca indique que la version n'a été rendu accessible qu'aux Bêtas testeurs. C'est une version sandbox qui permet de tester de nouvelles fonctionnalités.
  • Par le futur, ce sera indiqué par: Version Test une fois la fin de la Bêta

    • CORRECTION MAJEURE
  • Des erreurs critiques ont été détectées et l'application et ont été corrigées. L'application est indisponible temporairement ou l'accès est bloqué.

    • CORRECTION Mineure
  • Une très courte maintenance pour modifier une partie du programme et/ou corriger des bugs mineurs

    • Pré-Release
  • Pré-Version soit Version non officielle, elle ne sera pas installée sur le serveur pour éviter une synchronisation avec vos clients.
  • Malgré tout, si vous souhaitez accéder à cette version, elle sera simplement, un lien sera intégré dans le Devlog comme suit: Installer la Pré-Release ici:

    • Ancienne Version
  • Les versions possédant cette indication ne seront pas disponible à la Réinstallation.

    • Mise à Jour Client
  • Portant un numéro de version différent, cet indicateur précise que seul le serveur ou une donnée quelconque à été mise à jour mais qui n'impacte pas directement l'accès à la publication (exemple: modification de l'écran de chargement, de la synchro des données, des API's etc...)

    • + Sécurisée
  • Si la version la plus récente n'est pas la plus sécurisée, alors ce badge sera affiché. Les versions comportant ce badge OU incluant dans les devlogs le texte suivant: Cette version possède aussi l'indice: "+ Sécurisée"

    • Nouvelle Feature
  • Indique qu'une nouvelle fonctionnalité à été ajoutée à l'application.
